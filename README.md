# 船舶智能导航路径规划数据集可视化演示平台

本应用使用 `Streamlit + Folium` 将已有 AIS 清洗平滑、航道拓扑、动态边权和多目标路径规划成果组织为一个更接近“产品原型”的课程展示界面。

这次界面重构的目标不是改变算法，而是解决三个展示问题：

1. 避免 Streamlit 页面过于像“数据分析 demo”；
2. 通过主题、卡片、层级和首页叙事增强视觉品质；
3. 将导航系统从“控件堆叠”改为“项目总览 → 数据治理 → 航道网络 → 动态边权 → 智能规划 → 说明边界”的自然路径。

## 页面结构

- `项目总览`：首屏 Hero、核心成果指标、功能入口、技术链路与典型案例讲解。
- `智能规划`：保留原有起终点选择、船舶约束、A* / Dijkstra 规划、路线对比和导出功能。
- `数据治理`：解释 AIS 数据清洗、平滑、聚合与质量标注的作用。
- `航道网络`：展示节点、边、案例区域与路线差异，帮助理解水上路网结构。
- `动态边权`：展示耗时、通行次数、置信度等边权指标，解释路线推荐依据。
- `说明与边界`：说明数据口径、应用边界和课堂/比赛讲解话术。

## 安装与运行

在本目录执行：

```powershell
python -m pip install -r requirements.txt
python -m streamlit run app.py
```

浏览器打开 Streamlit 提示的本地地址即可进入界面。

如果希望不修改全局 Python 环境，也可将地图组件安装在项目本地目录；应用会自动加载 `.vendor`：

```powershell
python -m pip install --target .vendor folium streamlit-folium
python -m streamlit run app.py
```

## 推荐演示流程

1. 在 `项目总览` 介绍 AIS 数据到导航路线的完整成果链。
2. 在 `数据治理` 解释为什么需要清洗、平滑和质量标注。
3. 在 `航道网络` 与 `动态边权` 说明真实轨迹和置信度如何支撑选路。
4. 在 `智能规划` 选择 `西江内河` 或自定义起终点，同时开启三条路线，突出综合最优路线的可靠性提升。
5. 在 `说明与边界` 主动说明该系统是课程展示原型，不替代真实电子海图、桥梁净空、海事通告或现场调度决策。

## 数据与边界说明

- 现有交付物包含 `12,305` 个航道节点、`47,826` 条有向边和 `30,106` 条精细化统计边。
- 报告中的权重建模有效事件为 `241,224` 次；交付权重文件直接映射到路网后可汇总的有效通行次数为 `227,903` 次，应用中分别标注两种口径。
- 当前边表中的吃水和限高字段为约束机制演示占位值（默认 `20m / 50m`）。系统可以执行约束过滤，但结果不能替代真实电子海图、桥梁净空或海事通告。
- 本展示层读取既有成果，不重新运行原始 AIS 清洗过程，也不提供实时船舶在线导航。

## 设计改造说明

新增文件：

```text
app.py                    # 产品化首页
ui.py                     # 统一卡片、Hero、指标与 CSS 设计系统
.streamlit/config.toml    # Streamlit 主题配置
pages/1_智能规划.py       # 多页入口，复用原 smart_nav_app.py
pages/2_数据治理.py
pages/3_航道网络.py
pages/4_动态边权.py
pages/5_说明与边界.py
```

原 `smart_nav_app.py` 的算法调用、地图交互、路线导出逻辑保持不变，以降低重构风险。
